## AI Agent vs Chatbot

1. 定位不同
  - `Chatbot`：主要目标是进行自然语言对话，回答用户提问、保持聊天连贯性。
  - `AI Agent`：主要目标是“完成任务”，能规划步骤、必要时调用工具并对结果进行迭代直到达成目标。
2. 能否主动行动
  - `Chatbot`：通常以“生成回复”为主，更多是被动回应用户。
  - `AI Agent`：可以主动选择行动（例如检索、调用 API、执行代码、写文件）并把行动结果反馈回模型继续推理。
3. 工具使用与外部交互
  - `Chatbot`：工具使用不一定有，或者仅用于增强回复。
  - `AI Agent`：常见架构是“模型 + 工具/环境”，把外部能力当作可调用模块。
4. 迭代与闭环
  - `Chatbot`：通常是一轮或少量轮次的对话生成。
  - `AI Agent`：强调多轮闭环（观察 -> 推理/规划 -> 行动 -> 观察 -> …），直到任务完成或达到停止条件。
5. 状态与记忆
  - `Chatbot`：状态更多是对话上下文（短期为主）。
  - `AI Agent`：可能维护更明确的任务状态（已完成步骤、待办事项、工具调用结果、约束等）。

## AI Agent vs ReAct 循环

1. 概念层级不同
  - `AI Agent`：是更大的系统/方法论，包含任务目标、状态管理、工具调用、停止策略等。
  - `ReAct`：是一种常见的“执行/提示策略/推理格式”，把过程拆成“Reasoning(推理)”和“Act(行动)”并形成循环。
2. 作用方式
  - `ReAct`：通常在每轮让模型先给出推理，再决定要调用的工具/动作（或选择是否继续），并把动作结果作为下一轮的输入。
  - `AI Agent`：可以使用 ReAct，也可以使用其他策略（例如 Plan-and-Execute、Function Calling、工具路由、层级规划等）。
3. 重点不同
  - `ReAct`：重点在“如何让模型在每一步都能做出行动选择”，让推理过程和工具调用交织。
  - `AI Agent`：重点在“如何把这些步骤组织成可完成任务的闭环”，包括多工具、多步骤、失败恢复、成本控制、停止条件等。
4. 可替换性
  - `ReAct` 只是实现 agent 的一种手段或协议；
  - 换成其他循环/规划机制，agent 依然可能成立。
